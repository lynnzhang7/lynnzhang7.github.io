The Project

Holographic Cell Visualizer is a mixed reality HoloLens application for cancer researchers, developed by interns from the Microsoft Garage program. Our project, sponsored by BC Cancer, enables researchers to intuitively interact with and explore holograms of 3D genomic data.

Background

The Microsoft Garage program is a 16 week internship, where interns are typically put in a group of seven — five developers, one designer, one PM — to focus on a single project. Garage is a very unique internship because the intern team is driving the project: coming up with a vision, deciding on features, organizing sponsor meetings, planning sprints, all with limited supervision from our manager and our coaches. Most of these projects start from scratch, but my team had the unique opportunity to continue a project that was started by a previous intern group.

The project that we started on (video summary here) already had the major components of our app: the processing and rendering of the 50,000 cells, different modes to explore different aspects of the data, and the beginnings of a companion web application. When my intern group started, we reassessed the state of the project with our sponsors, and decided on the three key features we wanted to add.

Intuitive Interactions

One of the major benefits of HoloLens over other 3D imaging platforms is how intuitive it is to interact with. To take advantage of this, we wanted to add the ability to use HoloLens gestures to move the cell model.

Sticky Note Integration

Besides BC Cancer, our other sponsor was Microsoft's Sticky Notes team. Our goal was the integrate the Sticky Notes API into our app so that researchers could place holographic notes and access them from anywhere.

Jupyter Notebook API

Our most important goal was to find a way to integrate our application into a cancer researcher's workflow. We decided to focus on Jupyter Notebook since it is one of the most widely used tools for data analysis.



Besides the three goals listed above, one general goal we had was to increase the functionality of the companion web app. While the HoloLens is great at visualizing and interacting with data, it has limitations for small, precise actions. We assessed all the features we had and the new features we wanted to implement, and delegated aspects of them to either the HoloLens app or the web app based on which was more suitable.

We also wanted to implement some sort of saved state, so that researchers would be able to make some observations, and come back to the same sticky notes, same cell model orientation, etc. to continue observing or to share with a fellow researcher.

Final Product

By the end of the four months, my team had integrated gestures, created a Python API for Jupyter Notebook, and added the idea of sessions to the app. While we had implemented a way for researchers to annotate the cell model using a version of sticky notes, we were not able to integrate the Microsoft Sticky Notes API due to issues with authentication.

Some of the gestures we implemented included:

  • Air-tap and drag to rotate the model
  • Two-handed pinch and drag to resize the model
  • Double-air-tap to place a note

Our Python API enabled researchers to push cell groups — the name of the group and a list of cells that belong to that group — to the app at runtime. This feature allowed the most flexibility to the researcher: for example, if they wanted to see all the cell types in the app, they could push a list of cells for each cell type.

We wanted to take advantage of the web app as much as possible, since HoloLens was l imited at consistently detecting small motions such as typing, so we came up with the following pages:

  1. Dashboard page: where the user can move the cell model by clicking on the mouse transform box and switch between different modes
  2. Filter page: a list of all the cell groups, their colors and their visibility status
  3. Sticky page: any sticky notes created in HoloLens would show up here. From this page, they can be edited or deleted, with everything synced with the HoloLens app.
  4. Help page: some FAQs and helpful tips

Tech Stack

Both the HoloLens web app components were developed using Unity, with C# as the primary language. The web app and HoloLens communicated using Photon as a networking framework, and all the cell data and session data was stored in Azure. Our Jupyter Notebook API was written in Python.

My Role

My most important development contribution was to the Jupyter Notebook integration. After various user testing sessions with reseachers, I understood what their vision was and took the lead on creating a plan to integrate it into our application. This involved a very thorough understanding of the entire codebase, since this feature needed to fit within the constraints of the features we already had. With this plan, my team was able to finish the basis of this feature ahead of schedule.

In addition to being a developer, my unique role on the team was the UX champion. I worked closely with our designer to implement designs into both components of the application, and evaluated how the design aligned with the functionality. I was the primary contributor to the web application interface, and also implemented various aspects of the HoloLens interface.

Reflection

I learned so much more than I expected to during this internship. Tons of soft skills: our first three weeks were dedicated to planning, organizing, creating and presenting a vision. The importance of working in a team: I learned very quickly to rely on my teammates and ask for help when I needed it. Time management and prioritization: throughout the internship, we struggled with having too much we wanted to do and not enough time to do it. We had to cut a couple features which was difficult to do, but we were able to implement the core features that would make our app useful to a researcher.
